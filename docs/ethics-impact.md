# Ethics Impact Statement

## Project

MoveIt App

## Purpose

MoveIt App is a demo application created for learning software development, deployment, security, and maintenance practices.

## Stakeholders

| Stakeholder | Interest or Concern |
|---|---|
| Users | Need a simple and reliable application experience |
| Developer | Responsible for building, testing, and maintaining the project |
| Instructor | Reviews whether the project meets course requirements |
| Future maintainers | Need clear documentation and readable code |
| GitHub visitors | May view the project, license, and documentation |

## Potential Ethical Risks

| Risk | Possible Harm | Mitigation |
|---|---|---|
| Misleading users | Users may think the demo app is a finished production system | Clearly label the project as a demo or student project |
| Poor accessibility | Some users may have difficulty using the app | Use readable text, clear layout, and basic accessibility practices |
| Inaccurate documentation | Future users may misunderstand how the project works | Keep documentation updated with project changes |
| Security weakness | Users or maintainers may be exposed to unsafe practices | Use input validation, dependency audits, and avoid committing secrets |
| Data misuse | User information could be collected without clear purpose | Limit data collection and explain privacy practices |

## Professional Responsibilities

The developer should:

- Write clear and maintainable code
- Avoid copying work without attribution
- Protect sensitive values
- Document known issues
- Be honest about project limitations
- Respect user privacy

## Mitigation Summary

This project reduces ethical risk by documenting its purpose, using a license, providing privacy information, listing third-party libraries, and following basic secure coding practices.
