# QA Plan

## Test Levels
- Unit Testing: Test individual functions/components
- Integration Testing: Test interaction between frontend and backend
- System Testing: Test full application workflow

## Entry Criteria
- Code is complete
- Features are implemented
- No major errors

## Exit Criteria
- All tests pass
- No critical bugs (S1)
- System is stable

## Severity Levels
- S1 – Critical (system crash, cannot use system)
- S2 – Major (important feature broken)
- S3 – Minor (small issue, workaround exists)
- S4 – Low (UI issues, cosmetic)
